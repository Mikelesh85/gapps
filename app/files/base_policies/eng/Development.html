Purpose and Scope

  • This policy outlines the requisite measures for establishing and upholding foundational security standards for company software, network devices, servers, and desktop systems.
  • It applies to all individuals engaged in software development, system administration, and related activities within the organization, encompassing employees, contractors, and external parties involved in these functions.
  • This policy is accessible to all users within its scope.

Background

  • The underlying objective of this policy is to ensure a well-structured, secure, and consistent approach to managing the complete lifecycle of software and information systems.
  • It covers the entire spectrum, from initial requirements analysis to eventual system decommissioning.
  • The policy delineates procedures, roles, and responsibilities for each stage of the software development lifecycle.
  • The software development lifecycle may be executed in various forms, including linear progression or iterative cycles.

Policy

  • This policy outlines the key principles and actions governing the organization's software development lifecycle:
  • Software Development Life Cycle (SDLC): The organization's SDLC comprises the following phases:
  • Requirements Analysis
  • Architecture and Design
  • Testing
  • Deployment/Implementation
  • Operations/Maintenance
  • Decommission
    • Protection of Non-Production Data:
    • Throughout SDLC phases excluding production, live data sets containing personally identifiable information, financial data, or sensitive information must not be utilized.
    • Sensitive information is restricted from use beyond production environments.
  • Requirements Analysis Phase:
    • Analyze business requirements.
    • Perform risk assessment, guided by the Risk Assessment Policy (reference [a]).
    • Address security aspects (e.g., confidentiality, integrity, availability) relevant to the requirement.
    • Review regulatory mandates, organizational policies, standards, procedures, and guidelines.
    • Consider future business goals and current operational landscape.
    • Incorporate program management aspects.
    • Define security solution requirements.
    • Assess security solution costs and budget constraints.
    • Approve security requirements and budget.
    • Make informed decisions on "buy vs. build" for security services.
  • Architecture and Design Phase:
    • Educate development teams on secure system development.
    • Develop or refine infrastructure security architecture.
    • Specify technical and non-technical security controls.
    • Conduct architecture walkthrough.
    • Create system-level security design.
    • Formulate high-level technical and non-technical security designs.
    • Analyze design components through cost-benefit analysis.
    • Document detailed technical security design.
    • Perform comprehensive design reviews.
    • Define security processes, procedures, and segregation of duties.
    • Establish end-user training and awareness programs.
    • Formulate a general security test plan.
    • Update organizational policies, standards, and procedures as required.
    • Mitigate residual vulnerabilities within application and infrastructure.
  • Development Phase:
    • Configure secure development environment.
    • Provide training on software installation and configuration for infrastructure teams.
    • Develop code for application-level security.
    • Set up and integrate testing infrastructure.
    • Establish security-related vulnerability tracking processes.
    • Draft detailed security test plan and execute unit and integration testing.
  • Testing Phase:
    • Review code and configurations through static and dynamic analysis to identify vulnerabilities.
    • Test configuration procedures.
    • Perform system tests and assess performance under security controls.
    • Conduct usability testing of application security measures.
    • Execute independent vulnerability assessments on both infrastructure and application.
  • Deployment Phase:
    • Pilot deployment of infrastructure, application, and related components.
    • Transition from pilot to full-scale deployment.
    • Verify file integrity to ensure authenticity.
    • Conduct training and awareness programs for administrative personnel and users.
    • Require dual developer participation for production environment deployment.
  • Operations/Maintenance Phase:
    • User and access administration.
    • Performance tuning.
    • Scheduled backups in line with System Availability Policy.
    • Routine system maintenance, including security updates and patches.
    • Deliver training and awareness initiatives.
    • Conduct periodic system vulnerability assessments.
    • Perform annual risk assessments.
    • Validate operational system security controls and monitor logs.
    • Implement proactive system and user monitoring.
    • Execute software and hardware patching within specified timelines.
  • Decommission Phase:
    • Upon decommissioning, these steps are observed:
  • Conduct unit and integration testing following component removal.
  • Facilitate operational transition for component removal or replacement.
  • Define data retention requirements for software and system data.
  • Document detailed technical security design.
  • Update organizational policies, standards, and procedures as necessary.
  • Resolve remaining vulnerabilities within the application and infrastructure.
