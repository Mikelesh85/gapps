Purpose and Scope

·         This policy outlines the requisite measures for establishing and upholding foundational security standards for company software, network devices, servers, and desktop systems.

·         It applies to all individuals engaged in software development, system administration, and related activities within the organization, encompassing employees, contractors, and external parties involved in these functions.

·         This policy is accessible to all users within its scope.

Background

·         The underlying objective of this policy is to ensure a well-structured, secure, and consistent approach to managing the complete lifecycle of software and information systems.

·         It covers the entire spectrum, from initial requirements analysis to eventual system decommissioning.

·         The policy delineates procedures, roles, and responsibilities for each stage of the software development lifecycle.

·         The software development lifecycle may be executed in various forms, including linear progression or iterative cycles.

Policy

·         This policy outlines the key principles and actions governing the organization's software development lifecycle:

o   Software Development Life Cycle (SDLC): The organization's SDLC comprises the following phases:

§  Requirements Analysis

§  Architecture and Design

§  Testing

§  Deployment/Implementation

§  Operations/Maintenance

§  Decommission

·         Protection of Non-Production Data:

o   Throughout SDLC phases excluding production, live data sets containing personally identifiable information, financial data, or sensitive information must not be utilized.

o    Sensitive information is restricted from use beyond production environments.

·         Requirements Analysis Phase: During this phase, the following activities are conducted:

o   Analyze business requirements.

o   Perform risk assessment, guided by the Risk Assessment Policy (reference [a]).

o   Address security aspects (e.g., confidentiality, integrity, availability) relevant to the requirement.

o   Review regulatory mandates, organizational policies, standards, procedures, and guidelines.

o   Consider future business goals and current operational landscape.

o   Incorporate program management aspects.

o   Define security solution requirements.

o   Assess security solution costs and budget constraints.

o   Approve security requirements and budget.

o   Make informed decisions on "buy vs. build" for security services.

·         Architecture and Design Phase: During this phase, the following tasks are accomplished:

o   Educate development teams on secure system development.

o   Develop or refine infrastructure security architecture.

o   Specify technical and non-technical security controls.

o   Conduct architecture walkthrough.

o   Create system-level security design.

o   Formulate high-level technical and non-technical security designs.

o   Analyze design components through cost-benefit analysis.

o   Document detailed technical security design.

o   Perform comprehensive design reviews.

o   Define security processes, procedures, and segregation of duties.

o   Establish end-user training and awareness programs.

o   Formulate a general security test plan.

o   Update organizational policies, standards, and procedures as required.

o   Mitigate residual vulnerabilities within application and infrastructure.

·         Development Phase: During development, these tasks are completed:

o   Configure secure development environment.

o   Provide training on software installation and configuration for infrastructure teams.

o   Develop code for application-level security.

o   Set up and integrate testing infrastructure.

o   Establish security-related vulnerability tracking processes.

o   Draft detailed security test plan and execute unit and integration testing.

·         Testing Phase: During testing, these steps are taken:

o   Review code and configurations through static and dynamic analysis to identify vulnerabilities.

o   Test configuration procedures.

o   Perform system tests and assess performance under security controls.

o   Conduct usability testing of application security measures.

o   Execute independent vulnerability assessments on both infrastructure and application.

·         Deployment Phase: For deployment, the following activities occur:

o   Pilot deployment of infrastructure, application, and related components.

o   Transition from pilot to full-scale deployment.

o   Verify file integrity to ensure authenticity.

o   Conduct training and awareness programs for administrative personnel and users.

o   Require dual developer participation for production environment deployment.

·         Operations/Maintenance Phase: In this phase, ongoing operational security tasks include:

o   User and access administration.

o   Performance tuning.

o   Scheduled backups in line with System Availability Policy.

o   Routine system maintenance, including security updates and patches.

o   Deliver training and awareness initiatives.

o   Conduct periodic system vulnerability assessments.

o   Perform annual risk assessments.

o   Validate operational system security controls and monitor logs.

o   Implement proactive system and user monitoring.

o   Execute software and hardware patching within specified timelines.

·         Decommission Phase:

o   Upon decommissioning, these steps are observed:

§  Conduct unit and integration testing following component removal.

§  Facilitate operational transition for component removal or replacement.

§  Define data retention requirements for software and system data.

§  Document detailed technical security design.

§  Update organizational policies, standards, and procedures as necessary.

§  Address residual vulnerabilities in application and infrastructure.
